# Task Planner Dashboard

Многопользовательский планировщик задач с канбан-доской, таймером и календарем активности.

## Возможности

✅ **Канбан-доска** с 4 колонками приоритетов:
- 🔥 Срочная
- ⚡ Средняя срочность
- 📌 Малая срочность
- 🔮 В будущем

✅ **Многопользовательский режим**
- Регистрация и аутентификация
- Назначение задач участникам
- Отслеживание кто работает над задачей

✅ **Таймер отслеживания времени**
- Автоматический/ручной запуск (настройка в профиле)
- Отображение активного таймера на карточке
- История времени по задачам

✅ **Статусы задач**
- ✓ Выполнен
- ⏸ Отложен (с причиной)
- ▶ В работе

✅ **Система комментариев**
- Добавление комментариев к задачам
- Редактирование/удаление своих комментариев
- Real-time обновления

✅ **Календарь активности**
- Просмотр времени работы по датам
- Статистика по задачам
- Тепловая карта активности

✅ **Архив задач**
- Закрытие выполненных задач
- Просмотр архива

✅ **Real-time синхронизация**
- WebSocket для мгновенных обновлений
- Уведомления о назначении задач
- Уведомления о новых комментариях

✅ **Темная тема**
- Современный дизайн
- Адаптивная верстка

## Технологический стек

**Backend:**
- FastAPI (Python)
- SQLAlchemy 2.0
- SQLite
- JWT аутентификация
- WebSocket

**Frontend:**
- Vanilla JavaScript (ES6+)

## Установка и запуск

### 1. Backend

```bash
cd backend

# Создать виртуальное окружение
python -m venv venv

# Активировать (Windows)
venv\Scripts\activate

# Активировать (Linux/Mac)
source venv/bin/activate

# Установить зависимости
pip install -r requirements.txt

# Запустить сервер
python main.py
```

Backend будет доступен по адресу: `http://localhost:8000`

API документация: `http://localhost:8000/docs`

### 2. Frontend

```bash
cd frontend

# Запустить простой HTTP сервер
python -m http.server 8080
```

Frontend будет доступен по адресу: `http://localhost:8080`

## Использование

### Первый запуск

1. Откройте `http://localhost:8080` в браузере
2. Нажмите "Зарегистрироваться"
3. Введите имя пользователя, email и пароль
4. После регистрации вы автоматически войдете в систему

### Создание задачи

1. Нажмите кнопку "+ Создать задачу"
2. Введите название и описание
3. Выберите приоритет (колонку)
4. Нажмите "Сохранить"

### Работа с задачами

- **Взять задачу**: Нажмите "Взять" на карточке задачи
- **Запустить таймер**: Нажмите "▶ Старт" (автоматически запускается при взятии, если включено в настройках)
- **Остановить таймер**: Нажмите "⏸ Стоп"
- **Выполнить задачу**: Нажмите "✓ Выполнить"
- **Отложить задачу**: Нажмите "⏸ Отложить" и укажите причину
- **Комментарии**: Нажмите "💬 Комментарии"
- **Архивировать**: Нажмите "📦 Архив"
- **Перемещение**: Перетащите карточку в другую колонку

### Настройки

1. Нажмите "⚙️ Настройки" в шапке
2. Включите/выключите автоматический запуск таймера
3. Нажмите "Сохранить"

### Календарь

1. Нажмите "📅 Календарь" в шапке
2. Просмотрите активность по датам
3. Кликните на дату для просмотра деталей

### Архив

1. Нажмите "📦 Архив" в шапке
2. Просмотрите закрытые задачи

## API Endpoints

### Аутентификация
- `POST /auth/register` - Регистрация
- `POST /auth/login` - Вход

### Пользователи
- `GET /users/me` - Текущий пользователь
- `PUT /users/me/settings` - Обновить настройки

### Задачи
- `GET /tasks` - Список задач
- `POST /tasks` - Создать задачу
- `PUT /tasks/{id}` - Обновить задачу
- `DELETE /tasks/{id}` - Удалить задачу
- `POST /tasks/{id}/assign` - Назначить задачу
- `POST /tasks/{id}/complete` - Выполнить задачу
- `POST /tasks/{id}/postpone` - Отложить задачу
- `POST /tasks/{id}/archive` - Архивировать задачу
- `GET /tasks/archived/list` - Архив задач

### Таймер
- `POST /tasks/{id}/start-timer` - Запустить таймер
- `POST /tasks/{id}/stop-timer` - Остановить таймер

### Комментарии
- `GET /tasks/{id}/comments` - Список комментариев
- `POST /tasks/{id}/comments` - Добавить комментарий
- `PUT /comments/{id}` - Обновить комментарий
- `DELETE /comments/{id}` - Удалить комментарий

### Календарь
- `GET /calendar/sessions` - Сессии времени

### WebSocket
- `WS /ws` - Real-time обновления

## Troubleshooting

### Backend не запускается

```bash
# Проверьте версию Python (требуется 3.8+)
python --version

# Переустановите зависимости
pip install --upgrade -r requirements.txt
```

### Frontend не подключается к backend

1. Убедитесь, что backend запущен на `http://localhost:8000`
2. Проверьте консоль браузера на ошибки CORS
3. Убедитесь, что в `api.js` правильный `API_URL`

### WebSocket не работает

1. Убедитесь, что backend запущен
2. Проверьте консоль браузера на ошибки WebSocket
3. Попробуйте перезапустить backend
